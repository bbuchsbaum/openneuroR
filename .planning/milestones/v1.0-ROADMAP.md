# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-01-22
**Phases:** 1-5
**Total Plans:** 13

## Overview

This roadmap delivered an R package for programmatic OpenNeuro access in 5 phases. Started with package foundation and GraphQL-based discovery, built the download engine with HTTPS backend, added caching for persistence, layered in alternative backends (S3, DataLad) with the lazy handle pattern, and finished with testing infrastructure and polish.

## Milestone Summary

**Key Decisions:**
- httr2 direct (not ghql) — Simpler, fewer dependencies
- External .gql files — Maintainability, easier query updates
- S3 class (not R6) — Follows tidyverse patterns
- tools::R_user_dir for cache — CRAN-compliant
- DataLad > S3 > HTTPS priority — Integrity first, speed second
- httptest2 for mocking — Better httr2 integration

**Issues Resolved:**
- Search API returns null — Documented limitation, modality filter as alternative
- GraphQL filterBy.all is Boolean — Used search() query instead
- R version dependency — Added R >= 4.1.0 for |> pipe

**Issues Deferred:**
- Enhanced search filters (modality, species) — v2
- Concurrent multi-file downloads — v2
- targets integration (tar_openneuro) — v2

**Technical Debt Incurred:**
None — clean implementation throughout
