# Milestone v1.2: fMRIPrep Derivative Discovery

**Status:** SHIPPED 2026-01-23
**Phases:** 9-11
**Total Plans:** 7

## Overview

Enable researchers to discover and download fMRIPrep derivative datasets from OpenNeuro, including both embedded BIDS derivatives and the OpenNeuroDerivatives GitHub organization (780+ pre-computed fMRIPrep/MRIQC datasets).

## Phases

### Phase 9: Discovery Foundation
**Goal**: Users can discover available derivative datasets for any OpenNeuro dataset
**Depends on**: Phase 8 (v1.1 complete)
**Requirements**: DISC-01, DISC-02, DISC-03, DISC-04
**Success Criteria** (what must be TRUE):
  1. User can call `on_derivatives("ds000001")` and get a tibble of available pipelines
  2. Tibble includes pipeline name, source (embedded vs OpenNeuroDerivatives), and metadata
  3. OpenNeuroDerivatives GitHub organization repos are discoverable (784+ datasets)
  4. Discovery results are cached per-session (no repeated GitHub API calls within session)
**Plans**: 2 plans

Plans:
- [x] 09-01-PLAN.md — Session cache and GitHub API infrastructure
- [x] 09-02-PLAN.md — Main on_derivatives() function with embedded detection

**Completed:** 2026-01-23

### Phase 11: Download Integration
**Goal**: Users can download filtered derivative data with full test coverage

## Milestone Summary

**Key Decisions:**
- Closure-based session cache (avoids R namespace lock issues during runtime)
- Embedded derivatives preferred over GitHub when same pipeline exists in both sources
- Space matching is exact (prevents unexpected matches like "MNI" matching multiple spaces)
- Native space files always included when filtering (files without _space- entity are BIDS-compliant)
- Cache type field defaults to "raw" for backward compatibility

**Issues Resolved:**
- GitHub API rate limiting addressed via session caching (30 req/min throttle + retry logic)
- R namespace lock for cache worked around via closure pattern

**Issues Deferred:**
- MRIQC derivative support (v1.3+)
- Session/task filtering (v1.3+)
- FreeSurfer derivative support (v2+)

**Technical Debt Incurred:**
- None - all implementations complete with full test coverage
