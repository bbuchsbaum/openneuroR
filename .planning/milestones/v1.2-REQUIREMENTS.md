# Requirements: openneuro

**Defined:** 2026-01-20
**Core Value:** Researchers can find, download, and cache OpenNeuro datasets with a single pipeline-friendly API that just works.

## v1.2 Requirements (Complete)

Requirements for fMRIPrep Derivative Discovery milestone. Each maps to roadmap phases.

### Discovery

- [x] **DISC-01**: User can list available derivative pipelines for a dataset via `on_derivatives()` — SHIPPED
- [x] **DISC-02**: `on_derivatives()` returns tibble with pipeline name, source (embedded/OpenNeuroDerivatives), and metadata — SHIPPED
- [x] **DISC-03**: User can discover OpenNeuroDerivatives (784+ pre-computed fMRIPrep datasets from GitHub org) — SHIPPED
- [x] **DISC-04**: Discovery results are cached per-session to avoid GitHub API rate limits (60/hr unauthenticated) — SHIPPED

### Spaces

- [x] **SPAC-01**: User can list available output spaces for a derivative via `on_spaces()` — SHIPPED
- [x] **SPAC-02**: `on_spaces()` returns character vector of space names (MNI152NLin2009cAsym, T1w, fsaverage, etc.) — SHIPPED

### Download

- [x] **DOWN-01**: User can download fMRIPrep derivatives via `on_download_derivatives()` — SHIPPED
- [x] **DOWN-02**: `on_download_derivatives()` supports `subjects=` parameter for subject filtering (reuses v1.1 pattern) — SHIPPED
- [x] **DOWN-03**: `on_download_derivatives()` supports `space=` parameter for output space filtering — SHIPPED
- [x] **DOWN-04**: Downloaded derivatives use BIDS-compliant cache structure (`{dataset}/derivatives/{pipeline}/`) — SHIPPED

### Infrastructure

- [x] **INFR-01**: New functions have mocked tests (no real API/downloads in tests) — SHIPPED (36 tests added)
- [x] **INFR-02**: Package passes R CMD check with no errors/warnings after changes — SHIPPED (0 errors, 0 warnings, 0 notes)
- [x] **INFR-03**: S3 backend supports parameterized bucket for derivatives bucket access — SHIPPED
