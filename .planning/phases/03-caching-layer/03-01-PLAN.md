- Functions are internal (not exported), start with dot

Platform-appropriate paths will be:

<task type="auto">
  <name>Task 2: Manifest read/write with atomic writes</name>
  <files>R/cache-manifest.R, DESCRIPTION</files>
  <action>
1. Add jsonlite to DESCRIPTION Imports (was removed in Phase 2, needed for manifest I/O):
   - Add `jsonlite (>= 1.8.0)` to Imports section

2. Create R/cache-manifest.R with manifest operations:

`.manifest_path(dataset_dir)` - Returns path to manifest.json within dataset directory.

`.read_manifest(dataset_dir)` - Reads manifest.json if it exists, returns NULL if missing. Uses `jsonlite::read_json(path, simplifyVector = FALSE)`.

`.write_manifest(manifest, dataset_dir)` - Writes manifest to JSON using atomic pattern (temp file + fs::file_move). Use `jsonlite::write_json(manifest, temp_path, auto_unbox = TRUE, pretty = TRUE)` for human-readable output. Create dataset_dir if needed.

`.update_manifest(dataset_dir, new_file_info, dataset_id, snapshot_tag, backend)` - Reads existing manifest (or creates new), adds/updates file entry, writes back atomically. File entry structure:
```json
{
  "path": "sub-01/anat/T1w.nii.gz",
  "size": 15234567,
  "downloaded_at": "2026-01-21T10:30:00Z",
  "backend": "https"
}
```

`.new_manifest(dataset_id, snapshot_tag)` - Creates empty manifest structure:
```json
{
  "schema_version": 1,
  "dataset_id": "ds000001",
  "snapshot_tag": "1.0.0",
  "cached_at": "2026-01-21T10:30:00Z",
  "files": []
}
```

Error handling:
- Wrap write operations in tryCatch
- Clean up temp file on failure
- Use class "openneuro_cache_error" for cache-specific errors
- If manifest exists but is corrupt JSON, issue warning and treat as empty
  </action>

