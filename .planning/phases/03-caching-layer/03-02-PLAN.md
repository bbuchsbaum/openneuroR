<task type="auto">
  <name>Task 1: Integrate cache into on_download()</name>
  <files>R/download.R, R/download-progress.R</files>
  <action>
Modify on_download() to use cache by default and update manifest:

1. **Default dest_dir to cache:** When dest_dir is NULL, use `.on_dataset_cache_path(id)` instead of current working directory.

2. **Add use_cache parameter:** Add `use_cache = TRUE` parameter. When TRUE (default) AND dest_dir is NULL, use cache location. When FALSE or dest_dir is provided, use specified dest_dir.

3. **Check manifest for cached files:** Before downloading, read manifest with `.read_manifest()`. For each file in filtered_files, check if manifest contains entry with matching path AND the file exists with correct size. Skip these files (they're already cached).

4. **Update manifest after download:** In .download_with_progress(), after each successful file download, call `.update_manifest()` with file info (path, size, downloaded_at = ISO 8601 timestamp, backend = "https").

5. **Handle snapshot version:** Add snapshot_tag to manifest when caching. If user requests a different snapshot than what's cached:
   - Issue cli::cli_alert_info message: "Cache contains snapshot {cached}, downloading {requested}"
   - Proceed with download (will overwrite cached files)
   - Update manifest with new snapshot_tag

Update function signature:
```r
on_download <- function(id, tag = NULL, files = NULL, dest_dir = NULL,
                        use_cache = TRUE, quiet = FALSE, verbose = FALSE,
                        force = FALSE, client = NULL)
```

Update documentation:
- @param use_cache If TRUE (default) and dest_dir is NULL, downloads to CRAN-compliant cache location. Set FALSE to use current working directory.
- Update examples to show cache behavior
- Note that repeat calls skip already-cached files
  </action>
  <verify>
```r

```
