<task type="auto">
  <name>Task 1: Space extraction helpers and S3 file listing</name>
  <files>R/discovery-spaces.R</files>
  <action>
Create R/discovery-spaces.R with the following internal helpers:

1. `.extract_space_from_filename(filename)` - Extract space label from BIDS filename
   - Pattern: `_space-([A-Za-z0-9]+)` matches `_space-MNI152NLin2009cAsym_` etc.
   - Returns NA_character_ if no space entity found
   - Do NOT infer T1w from files without space entity (per BIDS convention, native space files often omit it)

2. `.extract_spaces_from_files(filenames)` - Extract unique spaces from filename vector
   - Apply `.extract_space_from_filename()` to each filename
   - Remove NAs, get unique values, sort alphabetically
   - Return character(0) if no spaces found

3. `.list_derivative_files_embedded(dataset_id, pipeline, tag, client)` - List files for embedded derivatives
   - Use on_files() to navigate to derivatives/{pipeline}/ tree
   - Recursively gather filenames from func/ and anat/ subdirectories
   - Sample approach: get first 2-3 subjects worth of files (don't need exhaustive listing)
   - Return character vector of filenames

4. `.list_derivative_files_s3(dataset_id, pipeline)` - List files from OpenNeuroDerivatives S3 bucket
   - Use processx to run: aws s3 ls --no-sign-request --recursive s3://openneuro-derivatives/{pipeline}/{dataset_id}-{pipeline}/ --max-items 500
   - Parse AWS ls output to extract filenames
   - Use .find_aws_cli() from backend-detect.R
   - Handle access denied gracefully (return empty vector, warn user)
   - Return character vector of filenames

Include roxygen2 documentation for all helpers (even internal @keywords internal).

  </verify>
