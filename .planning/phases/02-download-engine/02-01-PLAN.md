Create R/download-utils.R with these internal functions:

1. `.construct_download_url(dataset_id, file_path)` - Build S3 download URL
   - Base: https://s3.amazonaws.com/openneuro.org
   - URL-encode file_path with `utils::URLencode(file_path, reserved = TRUE)`
   - Return: "https://s3.amazonaws.com/openneuro.org/{dataset_id}/{encoded_path}"

2. `.download_atomic(url, final_path, ...)` - Download to temp, move on success
   - Create temp file with `fs::file_temp(ext = fs::path_ext(final_path))`
   - Call the download function (passed as `download_fn` param or internal)
   - On success: `fs::dir_create(fs::path_dir(final_path))`, then `fs::file_move(temp, final)`
   - On failure: delete temp file if exists, re-throw error
   - Handle cross-filesystem moves: try file_move, fallback to file_copy + file_delete

3. `.validate_existing_file(path, expected_size)` - Check if file exists with correct size
   - Return TRUE if file exists AND `fs::file_size(path) == expected_size`
   - Return FALSE otherwise

4. `.ensure_dest_dir(dest_dir, dataset_id)` - Set up destination directory
   - If dest_dir is NULL, use `fs::path(getwd(), dataset_id)`
   - Create directory with `fs::dir_create(dest_dir)`
   - Return absolute path

<task type="auto">
  <name>Task 2: Single-file download with progress, retry, resume</name>
  <files>R/download-file.R</files>
  <action>
Create R/download-file.R with these internal functions:

1. `.download_single_file(url, dest_path, expected_size = NULL, resume = TRUE, quiet = FALSE)` - Main download function
   - Check `interactive()` for progress bar display (suppress if non-interactive OR quiet=TRUE)
   - If resume=TRUE AND file >= 10MB AND partial file exists: call `.download_resumable()`
   - Otherwise: standard download with httr2 pipeline:
     ```r
     httr2::request(url) |>
       httr2::req_progress(type = "down") |>  # Only if !quiet && interactive()
       httr2::req_retry(
         max_tries = 3,
         retry_on_failure = TRUE,
         is_transient = \(resp) httr2::resp_status(resp) %in% c(429, 500, 502, 503, 504)
       ) |>
       httr2::req_perform(path = dest_path)
     ```
   - Return list(success = TRUE, path = dest_path, bytes = file_size)

2. `.download_resumable(url, dest_path, existing_bytes)` - Resume partial download
   - Add Range header: `httr2::req_headers("Range" = paste0("bytes=", existing_bytes, "-"))`
   - Perform request to temp file
   - Check response status:
     - 206 (Partial Content): Append temp content to existing file, delete temp
     - 200 (Full Content): Server ignored Range, replace existing file with temp
   - Return success indicator

3. `.get_file_size(path)` - Safe wrapper for fs::file_size
   - Return 0 if file doesn't exist
   - Return numeric size otherwise

Error handling:
- Wrap in tryCatch
- On httr2 error: clean up any partial files
- Re-throw with informative message: "Download failed for {basename(dest_path)}: {original_error}"

Add roxygen2 docs with @keywords internal.
  </action>
  <verify>devtools::load_all() succeeds; manual test downloading small file from S3 shows progress bar</verify>
  <done>Download function exists with httr2 progress, 3 retries with exponential backoff, and resume support for files >= 10MB</done>
</task>

<task type="auto">

  <action>
Create R/download-list.R with:

1. `.list_all_files(dataset_id, tag = NULL, client = NULL)` - Get all files recursively with full paths
   - Call `on_files(dataset_id, tag, client = client)` for root listing
   - Initialize result tibble with columns: filename, size, annexed, full_path
   - For each entry in root:
     - If directory=FALSE: add to result with full_path = filename
     - If directory=TRUE: recursively call `.list_directory()` and append results
   - Return tibble sorted by full_path

2. `.list_directory(dataset_id, tag, key, parent_path, client)` - Helper for recursive traversal
   - Call `on_files(dataset_id, tag, tree = key, client = client)`
   - For each entry:
     - Build full_path: `fs::path(parent_path, filename)`
     - If directory=FALSE: add to result
     - If directory=TRUE: recurse
   - Return tibble of files found

Result tibble structure:
```r
tibble::tibble(
  filename = character(),    # Just the filename
  full_path = character(),   # Full path from dataset root (e.g., "sub-01/anat/T1w.nii.gz")
  size = numeric(),          # Size in bytes
  annexed = logical()        # git-annex storage flag
)
```

Note: This function makes multiple API calls (one per directory). Add cli::cli_progress_step() for "Scanning files..." if interactive().
