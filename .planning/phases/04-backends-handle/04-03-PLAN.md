<task type="auto">
  <name>Task 1: Backend Dispatch Logic</name>
  <files>R/backend-dispatch.R</files>
  <action>
Create R/backend-dispatch.R with auto-select and dispatch:

1. `.select_backend(preferred = NULL)`:
   - If preferred is specified:
     - If .backend_status(preferred) is TRUE: return preferred
     - Else: warn "Requested backend '{preferred}' not available, falling back"
   - Auto-select by priority: c("datalad", "s3", "https")
   - Return first backend where .backend_status(backend) is TRUE
   - HTTPS is always available, so will always have a fallback

2. `.download_with_backend(dataset_id, dest_dir, files = NULL, backend = NULL, quiet = FALSE)`:
   - Select backend via .select_backend(backend)
   - If not quiet: cli::cli_alert_info("Using {selected} backend")
   - Execute download with tryCatch:
     - "datalad": .download_datalad(dataset_id, dest_dir, files, quiet)
     - "s3": .download_s3(dataset_id, dest_dir, files, quiet)
     - "https": Return NULL (signal to use existing HTTPS flow)
   - On openneuro_backend_error:
     - If selected != "https":
       - If not quiet: cli::cli_alert_warning("{selected} failed, falling back...")
       - Recursive call with next backend in priority
       - s3 falls back to https, datalad falls back to s3
     - Else: re-throw error
   - Return: list(success = TRUE, backend = selected)

3. Error handling in fallback:
   - Catch only "openneuro_backend_error" class
   - Let other errors propagate
   - Track original error for debugging if needed

Document with roxygen2 (@keywords internal).
  </action>

<task type="auto">
  <name>Task 2: Integrate Backend into on_download</name>
  <files>R/download.R, R/download-progress.R</files>
  <action>
Modify on_download() to support backend parameter:

1. Update on_download() signature:
   - Add `backend = NULL` parameter after `force`
   - Document: "@param backend Backend to use: 'datalad', 's3', or 'https'. NULL (default) auto-selects best available."

2. Update on_download() logic:
   - After setting up dest_dir, before calling .download_with_progress:
   - Call .select_backend(backend) to determine backend
   - If backend is "datalad" or "s3":
     - Call .download_with_backend(id, dest_dir, filtered_files$full_path, backend, quiet)
     - If success: Update manifest with backend field, print summary, return result
     - The S3/DataLad backends handle the actual download
   - If backend is "https" or backend dispatch returns NULL:
     - Continue with existing .download_with_progress() HTTPS flow

3. Update .download_with_progress():
   - Accept `backend` parameter (default "https")
   - Pass backend to .update_manifest() calls (already accepts backend param)

4. Update roxygen2 documentation for on_download()

The key insight: S3 and DataLad do their own file sync (they're smarter than file-by-file HTTPS). So when using those backends, we bypass .download_with_progress() and go directly to the backend, then record in manifest.
