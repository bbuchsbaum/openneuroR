  <action>
Create R/handle.R with lazy handle implementation:

1. `on_handle(dataset_id, tag = NULL, files = NULL, backend = NULL)`:
   - Validate dataset_id is non-empty character
   - Do NOT download anything - this is lazy
   - Return S3 object:
     ```r
     structure(
       list(
         dataset_id = dataset_id,
         tag = tag,
         files = files,
         backend = backend,
         state = "pending",  # "pending" | "ready"
         path = NULL,
         fetch_time = NULL
       ),
       class = c("openneuro_handle", "list")
     )
     ```
   - Export with @export

2. `on_fetch(handle, ...)` generic and method:
   - `on_fetch <- function(handle, ...) UseMethod("on_fetch")`
   - `on_fetch.openneuro_handle <- function(handle, quiet = FALSE, force = FALSE, ...)`:
     - If handle$state == "ready" && !force: return handle unchanged
     - Call on_download(id = handle$dataset_id, tag = handle$tag, files = handle$files, backend = handle$backend, quiet = quiet, force = force)
     - Update handle: state = "ready", path = result$dest_dir, fetch_time = Sys.time()
     - Return updated handle
   - IMPORTANT: Document that user must capture return value: `handle <- on_fetch(handle)`
   - Export with @export

3. `on_path(handle)` generic and method:
   - `on_path <- function(handle) UseMethod("on_path")`
   - `on_path.openneuro_handle <- function(handle)`:
     - If handle$state != "ready": abort with "Handle not yet fetched" and hint "Call on_fetch(handle) first"
     - Return handle$path
   - Export with @export

4. `print.openneuro_handle(x, ...)`:
   - Use cli for nice output:
     ```r
     cli::cli_text("{.cls openneuro_handle}")
     cli::cli_bullets(c(
       " " = "Dataset: {.val {x$dataset_id}}",
       " " = "Tag: {.val {x$tag %||% 'latest'}}",
       " " = "State: {.val {x$state}}",
       " " = if (x$state == 'ready') "Path: {.path {x$path}}" else "Path: <not fetched>"
     ))
     invisible(x)
     ```
   - Export with @export

5. All functions: Full roxygen2 documentation with @param, @return, @export, @examples

Reference RESEARCH.md Pattern 5 and Pitfall 5 for implementation details.
