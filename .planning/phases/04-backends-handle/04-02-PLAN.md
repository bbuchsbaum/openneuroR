  <action>
Create R/backend-datalad.R with DataLad download:

1. `.datalad_action(dest_dir)` - Determine clone vs update:
   - Check if fs::path(dest_dir, ".datalad") exists
   - If exists: return "update" (already a DataLad dataset)
   - If dest_dir exists and non-empty but no .datalad: abort with class "openneuro_backend_error" and message "Destination exists but is not a DataLad dataset"
   - Otherwise: return "clone"

2. `.download_datalad(dataset_id, dest_dir, files = NULL, quiet = FALSE, timeout = 1800)`:
   - Construct GitHub URL: `https://github.com/OpenNeuroDatasets/{dataset_id}.git`
   - Determine action via .datalad_action(dest_dir)

   - If action == "clone":
     - Run: `datalad clone {github_url} {dest_dir}`
     - processx::run(command = "datalad", args = c("clone", github_url, dest_dir), error_on_status = FALSE, timeout = 300)
     - Check result$status, abort on non-zero with stderr

   - Run datalad get (for both clone and update):
     - If files is NULL: get_args = c("get", ".")
     - Else: get_args = c("get", files)
     - processx::run(command = "datalad", args = get_args, wd = dest_dir, error_on_status = FALSE, timeout = timeout)
     - Check result$status, abort on non-zero with stderr

   - Return: list(success = TRUE, backend = "datalad")

3. Error handling:
   - Use rlang::abort() with class "openneuro_backend_error"
   - Include stderr in error message for debugging
   - Distinct error messages for clone failure vs get failure

4. Document with roxygen2 (@keywords internal)

Reference RESEARCH.md Pattern 3 and Pitfall 2 for exact implementation.
