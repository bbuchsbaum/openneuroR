Implement the core GraphQL infrastructure following patterns from RESEARCH.md:

1. **R/client.R** - Client configuration:
   ```r
   #' Create OpenNeuro API Client
   #'
   #' @param url API endpoint URL (default: OpenNeuro GraphQL endpoint)
   #' @param token API token (default: from OPENNEURO_API_KEY env var)
   #' @return An openneuro_client object
   #' @export
   on_client <- function(url = "https://openneuro.org/crn/graphql",
                         token = NULL) {
     token <- token %||% Sys.getenv("OPENNEURO_API_KEY", unset = NA)
     if (is.na(token)) token <- NULL

     structure(
       list(url = url, token = token),
       class = "openneuro_client"
     )
   }

   #' @export
   print.openneuro_client <- function(x, ...) {
     cli::cli_text("<openneuro_client>")
     cli::cli_text("URL: {.url {x$url}}")
     cli::cli_text("Authenticated: {.val {!is.null(x$token)}}")
     invisible(x)
   }
   ```

2. **R/graphql.R** - Query execution:
   - `.on_read_gql(name)` - Load query from inst/graphql/{name}.gql using system.file()
   - `on_request(query, variables, client)` - Execute GraphQL query:
     - Build request with httr2::request()
     - Set Content-Type: application/json
     - Set User-Agent: openneuro-r/{version}
     - Add body with query and variables
     - Add retry logic (httr2::req_retry, max_tries=3, transient errors 429/500/502/503)
     - Add rate limiting (httr2::req_throttle, 10/minute)
     - Add bearer auth if token present
     - Perform request and parse JSON
     - Check for GraphQL errors in response body (errors field)
     - Return data field
   - Use rlang::abort() with custom error classes for API errors

3. **R/utils-response.R** - Response parsing helpers:
   - `.null_to_na(x)` - Convert NULL to NA for safe extraction
   - `.parse_timestamp(x)` - Parse ISO timestamp to POSIXct
   - Placeholder for response parsing functions (will be used by API functions)

4. **GraphQL query files** in inst/graphql/:
   - **search_datasets.gql**: Query datasets with pagination, filtering
   - **get_dataset.gql**: Get single dataset by ID
   - **get_snapshots.gql**: Get snapshots for a dataset
   - **get_files.gql**: Get files for a snapshot (with tree parameter for subdirectories)

Use exact query structures from RESEARCH.md "Query Examples" section.

Important:
- Do NOT use ghql - use httr2 directly
- Handle GraphQL errors that come with HTTP 200 status
- All user-facing functions use snake_case naming
- Use rlang::%||% for null coalescing
